### 커밋 규칙
|태그 이름|설명|
|---|---|
|Feat|새로운 기능을 추가하는 경우|
|Fix|버그를 고친 경우|
|Style|코드 포맷 변경, 세미 콜론 누락, 코드 수정이 없는 경우|
|Refactor|리팩토링|
|Comment|필요한 주석 추가 및 변경|
|Docs|문서를 수정한 경우|
|Rename|파일 혹은 폴더명을 수정하거나 옮기는 작업만인 경우|
|Remove|파일을 삭제하는 작업만 수행한 경우|

`태그: 내용 (#이슈번호)`<br>

EX) Feat: 내용추가추가추가 (#1) , Refactor: Config 리팩토링 (#4)
